# MVP

The first useful version should be small enough to build, but real enough to test with an actual farm project.

## MVP thesis

Do not start by building a universal grant engine.

Start by proving that RegenOS can turn a restoration project into:

- a map-linked work plan
- a task list
- field evidence prompts
- an evidence pack

## First user

The first MVP user is a farmer or landowner.

This keeps the first version grounded in the person who has the land, the
problem, and the responsibility for deciding whether work is worth doing.
Advisers, contractors, and funders matter later, but the first loop should help
a farmer or landowner move from "something could be done here" to a practical,
evidenced work package.

## Suggested first project type

A small water restoration project:

- one farm
- several proposed ponds or check dams
- simple tree or hedge planting around them
- a limited set of evidence requirements
- one delivery team

## MVP features

### 1. Project record

Create a project with:

- name
- landowner/farm
- location
- intervention list
- funding source, if known
- people involved

### 2. Intervention templates

Use simple templates for:

- pond
- check dam / leaky dam
- tree planting
- hedge planting
- riparian fencing

Each template should define:

- typical tasks
- evidence requirements
- common risks
- materials
- equipment
- maintenance tasks

### 3. JobDone export

Generate a JobDone-compatible task package.

Each task should have:

- title
- description
- assignee
- due date or sequence
- location if relevant
- required evidence
- completion criteria

### 4. Evidence capture

For each task, define what evidence is needed:

- before photo
- during photo
- after photo
- GPS point
- measurement
- invoice
- sketch/map
- inspection note

### 5. Claim pack assembly

Produce a folder or document containing:

- summary of work completed
- task completion report
- evidence index
- photos
- invoices
- map references
- outstanding maintenance obligations

## What not to build first

- full grant scraping
- full international scheme database
- complex AI optimisation
- mobile app from scratch
- contractor marketplace
- carbon/biodiversity accounting

Those can come later. The first proof is task/evidence flow.

## First demo scenario

A farmer wants to create two ponds, install three small leaky dams, and plant trees around a wet corner of a field.

RegenOS should turn that into:

1. intervention plan
2. task list
3. evidence checklist
4. JobDone export
5. final evidence pack
